使用 3D Touch 让你的 APP 锦上添花

发表于 2015-11-17   |   分类于 Tech

简介

3D Touch 是苹果在 iPhone 6s/Plus 上引入的一种全新的交互方式。这也许是手机交互发展史上继苹果 2007 年在 WWDC 上发布第一代 iPhone 之后最大的革新了。当时苹果抛弃传统的全键盘手机，发布了一款带有电容式触摸屏支持多点触控的全触屏手机 iPhone。自此，全触屏手机席卷全球，到现在我们都还处于全触屏手机的风潮中，你很难再去买一块带键盘的手机了（而那些想要同时兼顾键盘和触屏的手机厂商已经死了。对，我就是在黑诺基亚，我不是人）。多点触控是全触屏手机能够流行的一大基石，用户很方便通过 pinch 来缩放一张图片，我 5 年前第一次在 iTouch（当时买不起 iPhone ）上放大一张图片那种操作快感带给我的喜悦我至今都记得。但在手机上无论是 pinch 还是滑动操作都是在一个二维平面上发生的，直到 iPhone 6s/Plus 的到来。新一代 iPhone 可以感知我们的压力，并且根据压力的不同给用户不用的反馈，其原理是在液晶屏的下面增加了一块压力传感器。相信我，如果你有体验过 iPhone 的 3D Touch 你一定会爱上她的，反正用了一个月 6s 后我已经离不开她了。

除了苹果自家的应用已经支持 3D Touch 之外，苹果也开放了以下 3D Touch API。

  • Home Screen quick action
  • UIKit peek and pop
  • Web view peek and pop
  • UITouch force properties

使用这些 API 可以让我们的 APP 在交互上更上一层楼！

Home Screen quick action

通过此 API 可以给应用程序的启动图标加上快速选项，用户可以通过快速选项菜单定位相应的功能。iOS 9 为我们提供了两种屏幕标签，Static quick actions 和 Dynamic quick actions。

Static quick actions 是在应用的 Info.plist 文件的 UIApplicationShortcutItems 数组中定义 静态快速选项。

Web view peek and pop

在 WKWebView 和 UIWebView 中苹果增加一个全新的属性 allowsLinkPreview 来开启 peek 和 pop，这样我们就很容易让我们自己的 web 拥有和 Safari 一样的 3D Touch 体验了。

UITouch force properties

iOS 9 中，UITouch 类增加了 force 和 maximumPossibleForce 两个属性来监测响应当前用户的触摸压力，force 实测范围在 0.0 ~ 6.6 之间。通过 force 属性我们可以突破 peek 和 pop 的限制，自定义 3D Touch！这个是我认为 iOS 9 给开发者带来的最大的惊喜。善用 force 属性可以赋予你的 APP 新的交互维度，让用户的操作更加符合直觉，提升体验。这里我简单的封装了一个 Button ，让它能够响应你的压力，随着你压力的增大或减小而缩放。具体的做法是在 UIButton 的子类中重写 touchesBegan touchesMoved touchesEnded 函数，然后使用 Closure 来回调触摸状态和 force。

总结

虽然还远没有达到交互革命这种地步，但 3D Touch 却是 iOS 交互史上的一个新篇章。虽然对用户来说有一定的学习成本，而且并非你 APP 所有的用户都是使用 iPhone 6s/Plus 。但是鉴于苹果在移动领域的霸主地位和无与伦比的推广能力，我相信用户使用 3D Touch 一定能够像他们使用重力感应，多点触控一样，成为交互的直觉，生活的必须。不要认为这个时间很遥远，回想一下苹果在 iPhone 5s 上第一次推出 Touch ID 指纹识别后，安卓阵营的各大厂商便争相推出带有指纹识别的手机，以至于到了今天指纹识别成了手机的标配（Windows Phone 除外）。这就是苹果的影响力，这就是苹果。永远走在科技和人文的十字路口，一直被模仿，从未被超越。
